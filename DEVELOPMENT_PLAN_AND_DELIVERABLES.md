# 舍友心晴开发计划与线上交付物

## 1. 项目概述

项目名称：舍友心晴：大学生宿舍压力预警与沟通演练助手

项目形式：网页版 AI 心理支持工具

技术路线：Vue + Python FastAPI + LangChain + SQLite / JSON

开发周期：6 天集中开发，第 7 天单独整理线上提交材料

团队成员：

| 成员 | 角色定位 | 主要负责内容 |
| --- | --- | --- |
| 朱春雯 | 组长、后端、AI、策划文档负责人 | 项目统筹、后端接口、压力评分模型、LangChain AI 模块、技术说明、安全边界、策划文档 |
| 曹乐 | 前端、UI、展示材料负责人 | Vue 前端页面、页面交互、UI 设计、截图素材、产品演示视频、宣传海报 |

开发目标：

- 完成一个可在线演示的网页 Demo。
- 跑通“宿舍事件记录 -> 压力分析 -> AI 沟通模拟 -> 沟通复盘报告”的核心流程。
- 准备线上提交所需的策划文档、演示视频、宣传海报、创意与场景说明。
- 产品定位保持非诊断性心理支持工具，不输出医学诊断或人格评价。

## 2. 核心功能范围

本项目优先完成以下核心功能：

| 功能模块 | 功能说明 | 是否必须完成 |
| --- | --- | --- |
| 首页 | 展示作品名称、产品定位、核心流程入口 | 必须 |
| 宿舍事件记录 | 收集事件类型、严重程度、发生频率、当前情绪、沟通状态、冲突升级情况、事件描述 | 必须 |
| 压力分析 | 根据事件信息计算压力值，返回风险等级、主要原因和建议 | 必须 |
| AI 沟通模拟 | 用户选择场景并输入话术，系统生成多个虚拟舍友回复 | 必须 |
| 沟通复盘报告 | 分析用户表达，输出优点、问题、优化话术、后续建议和安全提示 | 必须 |
| 历史记录 | 保存或展示历史事件记录 | 可选，时间不足时不作为核心验收项 |

- 开发优先级是核心闭环优先：事件记录 -> 压力分析 -> AI 沟通模拟 -> 沟通复盘报告。
- 第 1-6 天只处理产品开发、联调、文档和截图素材，不提前制作第 7 天的线上提交成品。
- 如果当前任务属于朱春雯范围，优先修改后端、AI、评分模型、接口、技术说明、策划文档。
- 如果当前任务属于曹乐范围，优先修改前端页面、UI、交互、截图素材和演示路径。
- 修改接口字段时，必须同步更新前端接口字段对照表和后端接口说明。
- 完成一个任务后，应记录对应交付物状态，避免后续重复开发。
