# A3-5 演示操作路径与录制说明

> 路径：`home -> record -> analysis -> simulate -> review`  
> 场景：使用**夜间噪音冲突**（典型噪音冲突案例）  
> 目标：稳定演示闭环；支持 `可无后端兜底`。

## A3-5 演示路径（稳定复现）

1. 首页（home）  
   - 打开首页，确认“记录”入口可见。  
2. 事件记录（record）  
   - 事件类型选择：`噪音冲突`  
   - 严重程度建议：`4`  
   - 发生频率建议：`每周多次`  
   - 当前情绪建议：`烦躁`  
   - 是否沟通：`否`  
   - 是否争吵/冷战：`是`  
   - 描述保持一句示例文本（可含“晚上打游戏声音影响作息”）  
3. 压力分析（analysis）  
   - 提交后进入分析页，确认压力分数、风险、来源与建议信息。  
4. 沟通模拟（simulate）  
   - 选择场景：`夜间噪音冲突`（标题区一致）  
   - 输入一句“你晚上能不能小声点，晚上我需要早点休息”  
   - 点击发送，确认三位角色回复有序出现  
5. 沟通复盘（review）  
   - 点击 `生成复盘报告`，确认“沟通复盘报告”完整生成（`表现总结`、`复盘维度`、`优化后的沟通话术`、`后续行动建议`、`安全提示`）。  

## A3-5 可无后端兜底策略

- 演示时如后端服务未启动、返回失败或字段不匹配：
  - `分析`页、`模拟`页、`复盘`页会使用各自的 `is_demo` 演示回复。  
  - 复盘页仍保持 `home -> ... -> review` 可达，不阻塞操作闭环。  

## 录制视频使用说明

1. 将以上 5 步串成一条 30~180 秒内的轨迹。  
2. 正式录屏只保留一条清晰主线；后端不可用演示兜底作为排练或备选分支检查，不必放进同一个视频。  
3. 文件命名建议：`demo-phase3-home-to-review.mp4`。  
4. 仅录制一次清晰演示，避免中间刷新页面以免切断链路。  

## 验收点

- A3-5 标准：路径是否稳定可复现，是否包含夜间噪音冲突案例，是否支持无后端兜底演示。
