# 页面设计严格对齐策划书设计说明

## 目标

在不修改原有 `frontend/页面设计/` 目录的前提下，新增一套严格对齐飞书策划书的静态页面设计稿。新设计用于后续迁移到 Vue 前端实现。

本次设计只解决页面内容与策划书一致性问题，不进入 Vue 实现，不更新截图文件。

## 范围

新增目录：

```text
frontend/页面设计-策划书对齐/
```

必须包含：

```text
首页/
事件记录/
压力分析/
沟通模拟/
沟通复盘报告/
shared/
```

五个页面目录均保留静态设计稿结构：

```text
DESIGN.md
code.html
```

不新增或更新 `screen.png`。

## 不改动内容

- 不修改 `frontend/页面设计/` 原始目录。
- 不更新原始页面截图。
- 不把本次设计直接迁移成 Vue 组件。
- 不加入心理诊断、人格评价、责任归因类文案。

## 视觉保留原则

新设计应以原 `frontend/页面设计/` 中对应页面为视觉基底，保留其 Playful Geometric 风格、侧边导航 / 底部导航结构、卡片布局、配色、字体、圆角、硬阴影和主要组件形态。除非为满足策划书字段、安全边界或流程完整性，否则不要重做页面布局或替换视觉风格。

## 页面设计

### 首页

首页沿用原首页的 Playful Geometric 视觉方向和主要布局，主信息改为更贴合策划书：

- 作品名称：舍友心晴。
- 定位：大学生宿舍压力预警与沟通演练助手。
- 主入口：开始记录宿舍事件。
- 展示宿舍压力晴雨表入口。
- 展示“记录 -> 分析 -> 演练 -> 复盘”的闭环。

首页首次进入时弹出一次安全说明弹窗。静态设计稿中直接展示弹窗态，并在页面上保留“安全说明”小入口，方便后续 Vue 实现中再次打开。

弹窗内容必须包含：

- 本工具仅用于宿舍压力趋势提示和沟通练习。
- 不进行心理疾病诊断。
- 不评价用户或舍友的人格、心理状态。
- 压力值不作为医学或心理诊断依据。
- 若出现高压力、严重冲突、持续失眠、强烈焦虑或暴力风险，建议联系辅导员、心理老师、家人或可信任同学。
- Demo 阶段不采集真实身份信息，演示数据使用虚拟样例。

首页原设计中“客观评估责任分配”等不符合边界的文案必须移除，替换为“识别压力来源，提供温和沟通建议”。

### 事件记录页

事件记录页继续采用“选择项 + 简短描述”的低门槛表单形式。字段必须覆盖策划书要求：

- 事件类型：作息冲突、卫生冲突、噪音冲突、费用冲突、隐私边界、情绪冲突。
- 严重程度：1-5 分。
- 发生频率：偶尔、每周多次、几乎每天。
- 当前情绪：烦躁、焦虑、委屈、愤怒、无奈、压抑。
- 是否已经沟通：是 / 否。
- 是否出现争吵或冷战：是 / 否。
- 事件描述：短文本输入。

提交按钮文案使用“保存并分析”，进入压力分析页。

### 压力分析页

压力分析页展示从事件记录得到的分析结果，示例数据以策划书中的夜间噪音冲突为主：

- 当前压力值：76 / 100。
- 风险等级：冲突风险较高。
- 主要压力来源：作息冲突、噪音问题。
- 情绪关键词：烦躁、压抑、无奈。
- 冲突风险趋势提示：问题发生频率较高，且尚未进行有效沟通，建议先进行沟通演练。
- 系统建议：选择双方情绪较平稳的时间进行现实沟通。
- 是否推荐进入 AI 沟通模拟：推荐。
- 安全提示：本结果仅用于压力趋势提示，不作为心理诊断依据。

页面保留“进入沟通演练”按钮。

### 沟通模拟页

沟通模拟页必须体现策划书中的“场景选择 + 多角色分别回应”。

场景选择至少包含：

- 舍友晚上打游戏太吵。
- 公共卫生长期无人打扫。
- 舍友未经允许使用私人物品。
- 水电费或公共费用分摊不均。
- 作息差异导致互相影响。
- 宿舍冷战或误会修复。

角色固定为：

- 舍友 A：直接型，容易反驳。
- 舍友 B：回避型，不愿正面沟通。
- 舍友 C：调和型，愿意缓和关系。

用户输入沟通话术后，页面展示三位 AI 舍友的分别回应，并展示系统提示：

```text
建议先表达感受，再提出具体请求。
```

静态设计稿使用示例输入：

```text
你晚上能不能小声点？
```

### 沟通复盘报告页

复盘报告页展示对用户表达方式的反馈。内容必须覆盖：

- 本次表达的优点。
- 可能引发防御心理的表述。
- 是否清楚表达了自己的感受。
- 是否提出了具体可执行请求。
- 是否给对方保留了沟通空间。
- 优化后的沟通话术。
- 后续行动建议。
- 安全提示。

示例优化话术采用策划书中的表达：

```text
我最近睡眠状态不太好，晚上声音比较容易影响我。我们能不能约定 12 点以后戴耳机？
```

安全提示：

```text
本建议仅用于沟通练习，不作为心理诊断依据。
```

## 组件与结构

新设计仍为静态 HTML，不要求拆成 Vue 组件。但 HTML 结构应为后续 Vue 迁移留出清晰边界：

- 导航区。
- 页面主标题区。
- 表单区或结果区。
- 安全提示区。
- 主操作按钮区。

`shared/` 放置可复用说明文件或片段：

```text
shared/safety-modal.html
shared/navigation-notes.md
```

如果实现时复制公共结构到各页面，也要保持导航文案一致。

## 数据流说明

静态设计稿只表达页面状态，不实现真实交互。后续 Vue 实现时的数据流为：

```text
首页 -> 事件记录 -> 压力分析 -> 沟通模拟 -> 沟通复盘报告
```

安全说明弹窗的后续实现行为：

- 首次进入网站时显示一次。
- 用户点击“我已了解，开始使用”后关闭。
- 后续可通过首页安全说明入口再次打开。

## 验收标准

设计稿通过以下检查即视为符合本阶段目标：

- 原 `frontend/页面设计/` 目录没有改动。
- 新设计目录存在，且包含五个核心页面。
- 首页包含首次安全说明弹窗设计。
- 事件记录页字段完整覆盖策划书字段。
- 压力分析页使用策划书的风险等级和 76 分示例。
- 沟通模拟页展示场景选择和 A/B/C 三角色分别回应。
- 复盘报告页包含优化话术、后续行动建议和安全提示。
- 页面文案不包含心理诊断、人格评价、责任归因。
- 新设计与原设计稿在整体视觉风格和主要布局上保持连续，只做策划书对齐所需的内容和局部结构调整。
- 不新增或更新任何 `screen.png`。
