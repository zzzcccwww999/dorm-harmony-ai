# Cao Le Phase 3 Frontend Closure Design

## Context

曹乐第三阶段任务来自 `DEVELOPMENT_PLAN_AND_DELIVERABLES.md` 6.2，范围是 A3-1 到 A3-5：修复前端流程问题、优化 UI 细节、检查响应式布局、整理页面截图素材、准备演示操作路径。当前 Vue 前端已经具备五个路由：首页、事件记录、压力分析、沟通模拟、沟通复盘；第二阶段 `verify:phase2` 和生产构建可通过。

本阶段不重做视觉风格，不新增第 7 天宣传海报或演示视频，不接管后端实现。第三阶段目标是把现有五页 Demo 收口成稳定可演示版本，并补齐验收脚本和演示材料。

## Requirements

- A3-1：`record -> analysis -> simulate -> review` 核心流程不能因空输入、后端不可用、缓存损坏或页面跳转状态中断。
- A3-2：首页、表单、压力分析卡片、聊天气泡和复盘报告应保持当前 Playful Geometric 视觉，并适合截图与视频展示。
- A3-3：桌面、平板和手机宽度下主内容、底部导航、表单选项、聊天回复和复盘卡片不重叠、不横向溢出。
- A3-4：整理首页、事件记录、压力分析、沟通模拟、沟通复盘五个页面的截图素材位置和用途。
- A3-5：提供一条稳定演示路径，默认使用典型夜间噪音冲突案例，可在无后端或后端异常时通过演示兜底跑完。

## Approach

本阶段采用最小收口方案。生产代码只做影响演示稳定性的前端调整：表单校验、缓存兜底、loading/error 文案、响应式约束和截图友好的布局细节。验证侧新增 `verify:phase3`，覆盖五页路由、核心流程状态、响应式 CSS 和第三阶段交付物文档；同时更新已过时的 `verify:week1`，避免旧三页断言误报。

演示材料放在 `frontend/docs/demo/`，包含问题修复记录、响应式检查记录、截图素材清单和操作路径说明。实际截图生成到 `frontend/docs/demo/screenshots/`，文件名与五个核心页面一一对应。

## Acceptance

- `npm run verify:week1` 通过，确认第一阶段核心三页仍可访问且布局基线有效。
- `npm run verify:phase2` 通过，确认第二阶段接口联调页面仍满足契约。
- `npm run verify:phase3` 通过，确认第三阶段收口交付物完整。
- `npm run build` 通过。
- 五个核心页面截图素材已生成或记录为可复现生成路径。
