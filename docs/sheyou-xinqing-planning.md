## 二、项目背景

大学宿舍是大学生学习生活中最重要的集体生活场景之一。宿舍成员长期共享空间，容易在作息、卫生、噪音、公共费用、私人物品、隐私边界等方面产生摩擦。多数宿舍矛盾并非一次性爆发，而是由小事件长期积累形成。例如，晚上打游戏声音过大、公共卫生分工不均、未经允许使用他人物品、生活习惯差异等问题，如果没有及时沟通，容易发展为冷战、争吵甚至更严重的人际冲突。

在现实场景中，很多学生并非不想解决问题，而是不知道如何开口。他们担心表达不满会破坏宿舍关系，也担心自己语气过重导致冲突升级。因此，宿舍矛盾往往呈现“有压力但不表达、有不满但选择忍耐、问题反复出现后突然爆发”的特点。

现有心理健康支持方式更多集中在心理咨询、辅导员谈话、心理健康课程等相对正式的场景中。对于日常宿舍关系压力，学生往往缺少低门槛、即时性、可练习的辅助工具。本作品希望通过“压力趋势提示 + AI 沟通演练”的方式，为大学生提供一种更轻量、更日常、更可操作的心理支持入口。

本作品不进行心理疾病诊断，不判断任何宿舍成员的人格或心理状态，而是聚焦于宿舍事件记录、压力趋势识别和沟通能力训练。

| 用户痛点 | 具体表现 | 对应需求 |
|-|-|-|

## 四、作品定位

“舍友心晴”是一款面向大学生宿舍关系的网页版 AI 心理支持工具。作品以宿舍冲突事件为切入点，通过用户记录宿舍生活中的矛盾事件，系统生成压力值、风险等级和主要压力来源；随后用户可以进入 AI 多角色沟通模拟场景，与不同性格的虚拟舍友进行对话演练；最后系统根据用户表达方式生成复盘报告和优化话术。

作品的核心目标不是替代心理咨询，也不是对用户或舍友进行心理诊断，而是帮助用户在真实沟通前进行低风险演练，降低表达焦虑，提升人际沟通能力，减少宿舍矛盾升级的可能性。

## 五、核心功能设计

### 5.1 宿舍事件记录

用户通过表单记录近期发生的宿舍事件。为了降低使用门槛，系统采用“选择项 + 简短描述”的方式，不要求用户填写长篇文字。

事件记录字段包括：

| 字段 | 示例 |
|-|-|
| 事件类型 | 作息冲突、卫生冲突、噪音冲突、费用冲突、隐私边界、情绪冲突 |
| 严重程度 | 1—5 分 |
| 发生频率 | 偶尔、每周多次、几乎每天 |
| 当前情绪 | 烦躁、焦虑、委屈、愤怒、无奈、压抑 |
| 是否已经沟通 | 是 / 否 |
| 是否出现争吵或冷战 | 是 / 否 |
| 事件描述 | “舍友晚上打游戏声音很大，影响睡眠。” |

### 5.2 压力晴雨表

系统根据用户提交的事件数据，生成当前宿舍关系压力值，并以图表方式展示。

压力值采用 0—100 分区间：

| 压力值 | 状态名称 | 说明 |
|-|-|-|
| 0—30 | 关系平稳 | 当前压力较低，可继续保持日常沟通 |
| 31—60 | 存在压力 | 宿舍关系存在一定摩擦，建议及时沟通 |
| 61—80 | 冲突风险较高 | 问题可能正在积累，建议进行沟通演练后再现实沟通 |
| 81—100 | 高压力状态 | 建议优先寻求现实支持，如辅导员、心理老师或可信任同学 |

压力晴雨表输出内容包括：

- 当前压力值；
- 风险等级；
- 主要矛盾来源；
- 情绪关键词；
- 系统建议；
- 是否推荐进入 AI 沟通模拟。

### 5.3 冲突风险趋势分析

系统不对未来事件作确定性预测，而是基于事件严重程度、发生频率、情绪强度、沟通状态和冲突升级情况，生成“冲突风险趋势提示”。

示例输出：

当前压力值为 76，处于“冲突风险较高”状态。主要压力来源为作息冲突和噪音问题，且该问题发生频率较高，用户尚未进行有效沟通。建议先通过 AI 沟通模拟练习表达方式，再选择舍友情绪较平稳的时间进行现实沟通。

### 5.4 AI 多角色沟通模拟

AI 沟通模拟是本作品的核心创新模块。系统通过 LangChain 对大模型调用流程进行组织，构建固定场景、固定角色、固定输出格式的多角色宿舍沟通模拟。

用户可选择典型宿舍场景，例如：

舍友晚上打游戏太吵；

公共卫生长期无人打扫；

舍友未经允许使用私人物品；

水电费或公共费用分摊不均；

作息差异导致互相影响；

宿舍冷战或误会修复。

系统设定多个 AI 舍友角色：

| 角色 | 性格设定 | 典型反应 |
|-|-|-|
| 舍友 A | 直接型，容易反驳 | “我也没多大声吧，你是不是太敏感了？” |
| 舍友 B | 回避型，不愿正面沟通 | “这个事情之后再说吧。” |
| 舍友 C | 调和型，愿意缓和关系 | “我们是不是可以一起定个规则？” |

用户输入自己的沟通话术后，AI 舍友会根据角色设定分别回应，推动对话发展。用户可在多轮对话中练习不同表达方式。

### 5.5 沟通复盘与话术建议

对话结束后，系统根据用户表达生成沟通复盘报告。

报告内容包括：

- 本次表达的优点；
- 可能引发防御心理的表述；
- 是否清楚表达了自己的感受；
- 是否提出了具体可执行请求；
- 是否给对方保留了沟通空间；
- 优化后的沟通话术；
- 后续行动建议。

示例：

你的表达能够明确指出问题，但语气较直接，容易让对方感到被指责。建议先描述自己的感受，再提出具体请求。可以改为：“我最近睡眠状态不太好，晚上声音比较容易影响我。我们能不能约定 12 点以后戴耳机？”

## 六、典型使用场景

### 场景一：晚上噪音冲突

用户最近多次被舍友夜间游戏声音影响睡眠，但担心直接提醒会引起不满。用户在系统中记录“作息冲突 / 噪音冲突”事件，系统生成压力值 76，并提示当前处于冲突风险较高状态。随后用户进入 AI 沟通模拟，与直接型、回避型、调和型舍友进行对话演练。系统最终生成更温和、具体的话术建议。

### 场景二：公共卫生分工不均

宿舍长期无人主动打扫公共区域，用户感到烦躁和委屈。系统识别该事件为“卫生冲突 + 高频重复 + 未有效沟通”，提示用户避免积压情绪，建议采用共同协商型表达，如“我们能不能重新排一个卫生轮值表，每个人负责固定时间？”

### 场景三：隐私边界冲突

舍友未经允许使用用户的私人物品，用户感到被冒犯但不知如何表达。系统推荐用户进入“个人边界表达”模拟场景，帮助用户练习既坚定又不攻击的表达方式，例如“这个东西对我来说比较私人，下次使用前可以先问我一下吗？”

## 七、创新点说明

### 7.1 从单纯情绪记录升级为宿舍关系压力趋势分析

传统情绪打卡工具通常只记录用户情绪变化，而本作品将情绪与具体宿舍事件关联，结合事件类型、发生频率、严重程度、沟通状态和冲突升级情况，生成宿舍关系压力晴雨表，使用户能够更直观地理解压力来源。

### 7.2 引入 AI 多角色宿舍沟通模拟

作品不是单纯提供建议，而是构建不同性格的 AI 舍友角色，让用户在真实沟通前进行模拟练习。多角色设定能够更接近真实宿舍沟通场景，使用户提前体验可能出现的反驳、回避、缓和等不同反馈。

### 7.3 形成“记录—分析—演练—复盘”的闭环

作品将事件记录、压力分析、AI 沟通模拟和复盘建议连接为完整流程。用户不仅能看到问题，还能获得可执行的表达方式和后续行动建议。

### 7.4 使用 LangChain 管理 AI 对话流程

后端通过 LangChain 对提示词模板、角色设定、输出格式和对话链路进行管理，提升 AI 模拟模块的结构化程度和可维护性。相比直接调用大模型接口，LangChain 更适合组织多步骤任务，如场景生成、角色回复、表达分析和复盘报告生成。

## 十、压力评分模型设计

本作品采用规则评分模型，不进行医学诊断，也不训练心理预测模型。评分结果仅用于宿舍关系压力趋势提示。

### 10.1 指标权重

| 指标 | 权重 | 说明 |
|-|-|-|
| 事件严重程度 | 30% | 事件对用户生活和情绪的影响程度 |
| 发生频率 | 25% | 同类问题是否反复出现 |
| 情绪强度 | 25% | 用户当前情绪的负面强度 |
| 沟通状态 | 10% | 是否已经进行过有效沟通 |
| 冲突升级情况 | 10% | 是否已经出现争吵、冷战或关系恶化 |

### 10.2 评分公式

压力值 = 严重程度得分 × 0.30
      \+ 发生频率得分 × 0.25
      \+ 情绪强度得分 × 0.25
      \+ 沟通状态得分 × 0.10
      \+ 冲突升级得分 × 0.10

各项得分统一换算为 0—100 分。最终输出压力值和对应风险等级。

### 10.3 输出示例

{
  "score": 76,
  "risk_level": "冲突风险较高",
  "main_reason": "作息冲突频繁出现，且尚未进行有效沟通。",
  "suggestion": "建议先通过 AI 沟通模拟练习表达方式，再选择合适时机进行现实沟通。",
  "disclaimer": "本结果仅用于压力趋势提示，不作为心理诊断依据。"
}

## 十一、LangChain AI 模块设计

### 11.1 模块目标

LangChain 模块用于组织大模型调用流程，使 AI 输出更加结构化、稳定和便于前端展示。该模块主要完成三类任务：

1. 根据宿舍事件生成沟通场景；
2. 根据角色设定生成多个 AI 舍友回复；
3. 根据对话记录生成沟通复盘报告。

### 11.2 Prompt 设计原则

| 原则 | 说明 |
|-|-|
| 固定角色 | 限定舍友 A、B、C 的性格和表达方式 |
| 固定场景 | 限定为大学宿舍关系，不扩展到其他心理诊断领域 |
| 固定输出格式 | 要求返回 JSON，方便前端解析 |
| 非诊断性 | 不输出心理疾病判断，不评价舍友人格 |
| 可操作性 | 建议必须具体、温和、可执行 |

### 11.3 多角色模拟输出格式

{
  "roommates": [
    {
      "name": "舍友A",
      "role": "直接型",
      "reply": "我也没多大声吧，你是不是太敏感了？"
    },
    {
      "name": "舍友B",
      "role": "回避型",
      "reply": "这个事情之后再说吧。"
    },
    {
      "name": "舍友C",
      "role": "调和型",
      "reply": "要不我们一起定一个晚上安静时间？"
    }
  ],
  "system_tip": "建议用户先表达感受，再提出具体请求。"
}

### 11.4 复盘报告输出格式

{
  "summary": "本次表达能够说明问题，但语气略直接。",
  "strength": "用户明确表达了自己的困扰。",

## 十五、风险控制与伦理说明

由于本作品属于心理健康相关方向，必须明确产品边界和安全原则。

### 15.1 非诊断性原则

本作品不进行心理疾病诊断，不判断用户或舍友是否存在心理问题，不输出医学结论。压力值仅用于宿舍关系压力趋势提示。

### 15.2 不评价他人原则

系统不会对舍友进行人格评价，也不会输出“某位舍友有问题”“某位舍友心理异常”等判断。系统只分析事件、表达方式和沟通策略。

### 15.3 安全求助提示

当压力值较高，或用户描述中出现严重冲突、持续失眠、强烈焦虑、暴力风险等情况时，系统提示用户及时联系辅导员、心理老师、家人或可信任的现实支持资源。

### 15.4 隐私保护原则

Demo 阶段不采集真实身份信息，演示数据使用虚拟样例。若后续扩展为正式产品，应提供数据加密、用户授权、数据删除和隐私声明机制。

### 15.5 AI 输出控制

通过固定 Prompt、固定输出格式和安全提示约束 AI 输出，避免生成攻击性建议、激化矛盾的表达或不恰当心理判断。

## 十七、演示脚本建议

1. 打开首页，介绍作品名称和定位：大学生宿舍压力预警与沟通演练助手。
2. 进入事件记录页，填写“舍友晚上打游戏声音太大”的事件。
3. 系统生成压力值 76，提示当前处于“冲突风险较高”状态。
4. 点击进入 AI 沟通模拟，选择“晚上噪音冲突”场景。
5. 用户输入一句较直接的话术：“你晚上能不能小声点？”
6. AI 舍友 A、B、C 分别给出不同反应。
7. 系统提示该表达可能略直接，并给出更温和话术。
8. 展示复盘报告，说明作品形成了“记录—分析—演练—复盘”的完整闭环。

## 十八、总结

“舍友心晴”聚焦大学生宿舍关系这一真实、高频、容易被忽视的心理压力场景。作品通过 Vue 前端、Python FastAPI 后端、LangChain AI 对话流程和规则评分模型，实现宿舍事件记录、压力趋势分析、多角色沟通模拟和复盘建议生成。相比普通情绪打卡工具，本作品更强调具体场景、真实互动和可执行建议，能够帮助大学生在低风险环境中练习沟通表达，缓解宿舍压力，减少冲突升级。

本作品边界清晰，不进行心理诊断，不替代专业咨询，适合作为心理健康教育、宿舍关系建设和班级心理工作中的辅助工具。
